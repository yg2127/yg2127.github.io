## 들어가며

문자열은 Python에서 정말 자주 쓰는 자료형인데, "수정이 안 된다"는 특성 때문에 처음에 좀 당황했다. 리스트처럼 `str[0] = 'x'` 하면 될 줄 알았는데 에러가 뜨더라. immutable이 뭔지 제대로 이해하고 나니까 왜 그런지 납득이 갔다.

## 핵심 개념

### 문자열 기초

- 문자열은 수정이 안 된다. C언어랑 똑같다
  - 이걸 'mutable하지 않다', 'immutable하다'고 한다. mutable은 list처럼 각 원소에 접근하고 수정할 수 있는 것을 말하는데, immutable한 string은 `str1 = 'abcdef'`일 때 `str1[3] = '2'` 이런 식의 수정이 안 된다
- 문자열 선언: `a = 'fdsaffdas'` 이런 식으로 선언한다
- `len(a)`하면 str 변수 a의 길이를 반환한다
- `\n` = 개행문자 (줄내림)
- `\t` = tab

### 문자열 연산

- 문자열 연산이 가능하다 (그대로 복붙한 게 반복되거나 이어붙여지거나(concat))
- 문자열 형변환이 가능하다 (`int("66")` → `66` 정수형 데이터로 변환된다)

### 슬라이싱

- `[이상:미만:간격]` 형식으로 슬라이싱한다

```python
a = 'abcdef'
a[::-1]  # 문자열 전체를 뒤에서부터 역순으로 출력한다
```

### 주요 메서드

- `upper()`, `lower()` → 대문자 변환, 소문자 변환
- `isupper()`, `islower()` → 대문자인지, 소문자인지 확인하여 반환

## 헷갈리기 쉬운 포인트

### 인덱싱 범위 — 왜 마지막이 포함 안 돼?

`a[1:3]`이 인덱스 1, 2만 포함하고 3은 빠진다. 처음엔 이게 직관적이지 않아서 헷갈렸는데, "이상:미만"이라고 외우면 된다. 이건 `range()`도 마찬가지다.

- `a[0:3]` → 인덱스 0, 1, 2 (3개)
- `a[:3]` → 처음부터 인덱스 2까지
- `a[3:]` → 인덱스 3부터 끝까지
- `a[-1]` → 마지막 문자

### immutable이라 수정하려면 새로 만들어야 한다

문자열의 한 글자만 바꾸고 싶어도 직접 수정은 안 된다. 대신 슬라이싱으로 잘라서 새 문자열을 만들어야 한다:

```python
s = "Hello"
# s[0] = 'h'  # TypeError 발생!
s = 'h' + s[1:]  # 'hello' — 새 문자열을 만들어서 재할당
```

### replace()를 몰라서 삽질하는 경우

문자열 안의 특정 부분을 바꾸고 싶으면 `replace()`를 쓰면 된다. 슬라이싱으로 복잡하게 할 필요 없이 `"Hello World".replace("World", "Python")` 하면 `"Hello Python"`이 나온다. 이것도 원본을 수정하는 게 아니라 새 문자열을 반환하는 거다.

## 정리하며

1. 문자열은 **immutable** — 수정 안 되고, 바꾸려면 새로 만들어야 한다
2. 슬라이싱은 **[이상:미만:간격]** — 끝 인덱스가 포함 안 되는 걸 항상 기억하자
3. `replace()`를 알면 문자열 수정 삽질을 줄일 수 있다
