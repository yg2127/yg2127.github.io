## 들어가며

클래스는 Python에서 OOP(객체지향 프로그래밍)의 시작점인데, 솔직히 처음에 "왜 이렇게 복잡하게 해야 하지?"라는 생각이 들었다. 특히 **self**가 왜 매번 들어가야 하는지가 전혀 납득이 안 갔다. 함수만으로 다 되는 것 같은데 클래스가 왜 필요한지는 코드가 복잡해지고 나서야 이해가 됐다.

---

## 핵심 개념

### 클래스와 인스턴스

- **클래스** = 붕어빵을 찍어내는 기계. 객체의 공통 속성, 개별 속성, 메서드를 정의한다
- **인스턴스** = 붕어빵. 클래스를 통해 생성된 실제 사용 가능한 객체다
- 객체는 모든 변수를 의미하고, 인스턴스는 클래스를 통해 생성된 객체를 의미한다

```python
class Dog(object):

    species = 'dog'  # 모든 인스턴스의 공통 속성

    def __init__(self, name, age):  # 개별 인스턴스의 속성 정의
        self.name = name
        self.age = age

    def bark(self):
        print(f'{self.name} is barking!')
```

### __init__ 메서드

`__init__` 메서드를 통해 인스턴스의 개별 속성을 정의한다. 입력값으로 `self`가 반드시 들어가야 한다.

```python
a = Dog('micky', 2)  # Dog 클래스의 인스턴스 a를 생성
```

### 인스턴스 활용

- `instance.__dict__`를 통해서 해당 인스턴스의 개별 속성을 dict 형식으로 반환받을 수 있다
- 생성된 인스턴스가 클래스 메서드를 실행하려면:

```python
a.bark()       # 'micky is barking!' 출력
Dog.bark(a)    # 클래스.메서드(인스턴스) 형식으로도 똑같이 실행된다
```

메서드를 선언할 때 `(self)`를 안 넣으면 그 메서드는 개별 인스턴스를 인풋으로 받지 않는다. 그 경우 `Dog.bark()` 형태로 호출해야 한다.

## 헷갈리기 쉬운 포인트

### self는 뭐고 왜 매번 써야 하나?

`self`는 "이 메서드를 호출하는 인스턴스 자기 자신"을 가리킨다. `a.bark()`를 호출하면 Python이 내부적으로 `Dog.bark(a)`를 실행한다. 즉 self 자리에 a가 들어가는 거다. self가 있어야 인스턴스의 속성(`self.name`)에 접근할 수 있다.

다른 언어(Java 등)에서는 `this`가 자동으로 전달되는데, Python은 명시적으로 써야 한다.

### 클래스 변수 vs 인스턴스 변수

```python
class Dog:
    species = 'dog'  # 클래스 변수 — 모든 인스턴스가 공유

    def __init__(self, name):
        self.name = name  # 인스턴스 변수 — 각 인스턴스마다 다름
```

`species`는 `Dog.species`로도, `a.species`로도 접근 가능하다. 하지만 `a.species = 'cat'`으로 바꾸면 a만 바뀌고 다른 인스턴스는 그대로다. 클래스 변수와 같은 이름의 인스턴스 변수가 생기면 인스턴스 변수가 우선된다.

### __init__은 생성자? 초기화?

엄밀히 말하면 `__init__`은 **초기화 메서드**지 생성자가 아니다. 객체 생성은 `__new__`가 담당한다. 하지만 실무에서는 거의 `__init__`만 쓰고, "생성자"라고 불러도 의사소통에 문제는 없다.

### 절차지향으로 다 되는데 왜 클래스를?

코드가 작을 때는 함수만으로 충분한데, 프로그램이 커지면 관련된 데이터(속성)와 행동(메서드)을 하나로 묶는 게 관리하기 훨씬 편하다. 예를 들어 "강아지"와 관련된 모든 걸 Dog 클래스에 모아두면, 나중에 강아지 관련 코드를 수정할 때 한 곳만 보면 된다.

---

## 정리하며

1. **self**는 인스턴스 자기 자신 — 메서드에서 속성에 접근하려면 반드시 필요하다
2. 클래스 변수는 공유, 인스턴스 변수는 개별 — `self.`이 붙으면 인스턴스 변수다
3. `__init__`은 인스턴스 생성 시 자동 호출되는 초기화 메서드
4. OOP의 장점은 코드가 커질수록 드러난다
