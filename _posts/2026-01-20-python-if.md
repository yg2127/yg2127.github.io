## 들어가며

조건문은 프로그래밍의 가장 기본적인 흐름 제어인데, Python의 조건문에는 다른 언어와 다른 독특한 점이 있다. 특히 truthy/falsy 개념을 모르면 "왜 이게 True야?" "왜 이게 False야?" 하면서 혼란에 빠진다.

## 핵심 개념

### 조건문 기초

- `if`-`elif`-`else` 문을 사용한다
- `and`, `or`, `not`을 사용한다
- `in`, `not in`을 야무지게 사용하도록 하자

```python
a = [1, 2, 3, 4, 5]
b = 7
if b not in a:
    print(f'{b} is not in a ({a})')
else:
    print(f'{b} is in a ({a})')
```

## 헷갈리기 쉬운 포인트

### truthy와 falsy — Python은 뭐든 조건이 된다

Python에서는 bool이 아닌 값도 조건문에 넣을 수 있다. 이게 처음엔 헷갈리는데, 규칙은 간단하다:

**Falsy (False로 취급되는 것들):**
- `False`, `None`
- 숫자 `0`, `0.0`
- 빈 자료형: `""`, `[]`, `()`, `{}`, `set()`

**나머지는 전부 Truthy (True)**

그래서 이런 코드가 가능하다:

```python
my_list = []
if my_list:  # 빈 리스트는 Falsy → False
    print("리스트에 데이터 있음")
else:
    print("리스트가 비어있음")
```

`if len(my_list) > 0:` 대신 `if my_list:` 이렇게 쓰는 게 더 Pythonic하다.

### 조건문 순서가 중요하다 — elif의 함정

`if`-`elif`는 위에서부터 순서대로 검사하고, **하나라도 참이면 나머지는 검사하지 않는다**. 그래서 조건이 겹칠 수 있는 경우 순서를 잘 정해야 한다:

```python
score = 85

# 잘못된 순서 — score >= 60이 먼저 참이 되어버린다
if score >= 60:
    grade = "C"
elif score >= 80:
    grade = "B"  # 여기 절대 도달 안 함!

# 올바른 순서 — 좁은 범위부터
if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 60:
    grade = "C"
```

### 삼항 연산자 — 한 줄 조건문

간단한 조건은 한 줄로 쓸 수 있다:

```python
age = 20
status = "성인" if age >= 18 else "미성년"
```

다른 언어의 `조건 ? 참 : 거짓`과 같은 건데, Python은 영어 문장처럼 읽힌다는 게 특징이다.

### and/or의 단축 평가

- `and`는 앞이 False면 뒤를 검사하지 않는다
- `or`는 앞이 True면 뒤를 검사하지 않는다

이걸 **단축 평가(short-circuit evaluation)**라고 하는데, 이걸 이용하면 에러를 방지할 수 있다:

```python
# my_dict에 'key'가 있을 때만 값을 검사
if 'key' in my_dict and my_dict['key'] > 10:
    print("조건 충족")
```

앞에서 `'key' in my_dict`가 False면 뒤의 `my_dict['key']`는 실행되지 않으므로 KeyError가 안 난다.

## 정리하며

1. **truthy/falsy** 알아두자 — 빈 자료형과 0은 False, 나머지는 True
2. `elif` 순서는 **좁은 범위부터** — 넓은 조건이 먼저 오면 아래가 실행 안 된다
3. 삼항 연산자: `값1 if 조건 else 값2` — 간단한 조건은 한 줄로
4. 단축 평가를 활용하면 에러 방지에 유용하다
