## 들어가며

집합(Set)은 수학 시간에 배운 그 집합이랑 거의 같다. 교집합, 합집합, 차집합 다 된다. 근데 처음에 "순서가 없다"는 게 뭔 소리인지 잘 와닿지 않았다. 인덱싱이 안 된다는 뜻이었다. `set[0]` 이런 거 안 된다.

## 핵심 개념

### 집합 선언

- 집합은 순서가 없고 중복이 되지 않는다

```python
b = set([1, 2, 3, 4, 4, 4, 4])
e = {1, 2, 3, ('f', 'i', 'x'), False, '~dict', 'set'}
```

### 집합 연산

- `a & b` = a, b의 교집합
- `a | b` = a, b의 합집합
- `a - b` = a, b의 차집합

### 주요 메서드

- `len()` → 길이 출력
- `isdisjoint()` → 중복 원소가 있는지(독립인지) bool 형식 반환
- `issubset()` → 부분집합인지 bool 형식 반환
- `add()` → 원소 추가
- `remove()` → 원소 제거
- `discard()` → 원소 제거
- `clear()` → 모든 원소 제거

## 헷갈리기 쉬운 포인트

### "순서가 없다"의 진짜 의미

순서가 없다는 건 인덱싱(`set[0]`)과 슬라이싱(`set[1:3]`)이 안 된다는 뜻이다. 출력할 때마다 순서가 달라질 수도 있다. 그래서 순서가 중요한 데이터는 set에 넣으면 안 된다.

### 중복 제거 트릭 — 이거 진짜 자주 쓴다

리스트에서 중복을 제거하고 싶으면 set으로 변환했다가 다시 list로 바꾸면 된다:

```python
numbers = [1, 2, 2, 3, 3, 3, 4]
unique = list(set(numbers))  # [1, 2, 3, 4]
```

단, **원래 순서가 보장되지 않는다**는 점에 주의. 순서를 유지하면서 중복을 제거하고 싶으면 `dict.fromkeys()`를 쓰는 방법이 있다.

### remove() vs discard()

둘 다 원소를 제거하는데 차이가 있다:
- `remove(x)`: x가 없으면 **KeyError** 발생
- `discard(x)`: x가 없어도 **에러 없이** 그냥 넘어간다

안전하게 제거하고 싶으면 `discard()`를 쓰자.

### 빈 set 만들기

`a = {}`는 빈 set이 아니라 **빈 딕셔너리**다! 빈 set을 만들려면 `a = set()`을 써야 한다. 이거 은근 실수하기 쉬운 부분이다.

## 정리하며

1. Set은 **순서 없음 + 중복 불가** — 인덱싱 안 된다
2. 리스트 중복 제거: `list(set(리스트))` — 순서는 보장 안 됨
3. `remove()`는 에러 발생, `discard()`는 안전 — 상황에 맞게 쓰자
4. `{}`는 빈 딕셔너리다 — 빈 set은 `set()`
