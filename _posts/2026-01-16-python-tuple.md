## 들어가며

튜플을 처음 배울 때 솔직히 "리스트가 있는데 왜 이게 필요하지?"라고 생각했다. 수정도 안 되고 삭제도 안 되는데 뭐하러 쓰나 싶었는데, 알고 보면 튜플이 쓰이는 이유가 분명히 있다.

## 핵심 개념

### 튜플 기초

- 수정이나 삭제가 안 되는 리스트이다
- 선언: `a = (1, 2, 3)` 이렇게 소괄호로 선언한다
- 인덱싱, 슬라이싱 모두 리스트와 똑같다
- 곱하기, 더하기 연산도 똑같다

## 헷갈리기 쉬운 포인트

### 왜 불변 리스트가 필요할까?

튜플이 존재하는 이유는 크게 세 가지다:

1. **안전성**: 실수로 값이 바뀌면 안 되는 데이터를 보호한다. 예를 들어 좌표 `(x, y)`, RGB 색상 `(255, 0, 0)` 같은 건 바뀌면 안 되니까
2. **딕셔너리 key로 쓸 수 있다**: 리스트는 mutable이라 딕셔너리 key가 될 수 없지만, 튜플은 immutable이라 가능하다
3. **성능**: 튜플이 리스트보다 메모리를 적게 쓰고 처리 속도도 빠르다

### 튜플 언패킹 — 이게 진짜 유용하다

튜플의 진가는 언패킹에 있다. Python에서 정말 많이 쓰이는 문법이다:

```python
# 기본 언패킹
a, b, c = (1, 2, 3)

# 변수 스왑 — 다른 언어에서는 temp 변수가 필요하지만 Python은 이렇게 된다
a, b = b, a

# 함수에서 여러 값 반환할 때도 사실 튜플이다
def get_name():
    return "Kim", 25  # 이게 튜플 ("Kim", 25)을 반환하는 거다
name, age = get_name()
```

### 원소가 하나인 튜플 주의

`a = (1)`은 튜플이 아니라 그냥 정수 `1`이다. 원소가 하나인 튜플을 만들려면 `a = (1,)` 이렇게 콤마를 꼭 찍어야 한다. 이거 모르면 나중에 `for`문 돌릴 때 에러나는 원인을 찾기 어렵다.

## 정리하며

1. 튜플은 "바뀌면 안 되는 데이터"를 위한 자료형이다
2. 딕셔너리 key로 쓸 수 있다는 게 리스트와의 핵심 차이
3. 언패킹 문법은 Python의 꽃이다 — 변수 스왑, 함수 반환값 등에서 자주 쓴다
4. 원소 하나짜리 튜플은 `(1,)` — 콤마 잊지 말자
